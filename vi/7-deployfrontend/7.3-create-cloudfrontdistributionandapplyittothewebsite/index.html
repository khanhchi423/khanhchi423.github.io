Tạo CloudFront Distribution và Áp dụng cho Website

Tạo CloudFront Distribution để Phân Phối Nội Dung Toàn Cầu

CloudFront là mạng phân phối nội dung (CDN) của Amazon giúp tăng tốc phân phối nội dung web tĩnh và động của bạn tới người dùng trên toàn thế giới. Trong phần này, chúng ta sẽ tạo một CloudFront distribution cho ứng dụng React của mình.

Bước 1: Truy cập Dịch vụ CloudFront

  1. Điều hướng đến AWS Management Console và tìm kiếm “CloudFront” trong thanh tìm kiếm.
  2. Nhấp vào CloudFront để mở bảng điều khiển dịch vụ.
  3. Nhấp Create a CloudFront distribution để bắt đầu quá trình thiết lập.

Bước 2: Cấu hình Distribution

Tùy chọn Distribution

  1. Distribution name: Nhập tên mô tả cho distribution của bạn: fastfood-react-app
  2. Description (tùy chọn): Thêm mô tả ngắn gọn: FastFood Order System - React App
  3. Distribution type: Chọn Single website or app vì chúng ta đang triển khai một ứng dụng React duy nhất.

Tên miền tùy chỉnh (Optional)
Nếu bạn có tên miền riêng, bạn có thể cấu hình ở đây. Nếu không, hãy để trống để sử dụng tên miền mặc định của CloudFront.

  • Domain (tùy chọn): Để trống hoặc nhập tên miền tùy chỉnh nếu bạn có.
  • Nhấp Check domain nếu bạn đã nhập tên miền để kiểm tra tính khả dụng.

Bước 3: Chỉ định Cấu hình Origin

Origin là nơi CloudFront lấy nội dung của bạn — trong trường hợp này là S3 bucket.

  1. Origin type: Chọn Amazon S3 vì chúng ta sẽ phục vụ nội dung từ S3 bucket.
  2. S3 origin: Nhập endpoint của S3 bucket:
    event-sourcing-fastfood-frontend-s3-website-ap-southeast-1.amazonaws.com
  3. Origin path (tùy chọn): Giữ nguyên /path hoặc chỉnh sửa nếu nội dung của bạn nằm trong thư mục con.

Bước 4: Cấu hình Bảo mật

Web Application Firewall (WAF)
Trong hướng dẫn này, chúng ta sẽ tắt WAF để giảm chi phí:

  1. Web Application Firewall (WAF): Chọn Do not enable security protections.
  2. Tùy chọn này phù hợp cho môi trường phát triển và thử nghiệm. Với ứng dụng sản xuất, hãy cân nhắc bật WAF để tăng cường bảo mật.

Bước 5: Xem lại và Tạo Distribution

  1. Xem lại toàn bộ cấu hình để đảm bảo chính xác.
  2. Nhấp Next để tiếp tục sang trang xem lại cuối cùng.
  3. Nhấp Create distribution để triển khai CloudFront distribution.

Quá trình triển khai CloudFront distribution thường mất 15–20 phút. Trạng thái sẽ chuyển từ Deploying sang Enabled khi hoàn tất.

Cập nhật Bucket Policy của S3

  1. Quay lại S3 bucket: event-sourcing-fastfood-frontend
  2. Vào tab Permissions
  3. Nhấp vào Bucket policy để chỉnh sửa policy hiện tại
  4. Cập nhật policy để bao gồm quyền truy cập từ CloudFront
    Policy sau khi cập nhật sẽ như sau:
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::event-sourcing-fastfood-frontend/*"
        },
        {
            "Sid": "AllowCloudFrontServicePrincipal",
            "Effect": "Allow",
            "Principal": {
                "Service": "cloudfront.amazonaws.com"
            },
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::event-sourcing-fastfood-frontend/*",
            "Condition": {
                "StringEquals": {
                    "AWS:SourceArn": "arn:aws:cloudfront::218453064234:distribution/E2MQTT186060K"
                }
            }
        }
    ]
}
  1. Nhấp Save changes để áp dụng policy đã cập nhật

Truy cập Ứng dụng của Bạn qua CloudFront

Khi CloudFront distribution đã được triển khai và bucket policy đã được cập nhật, bạn có thể truy cập ứng dụng React thông qua URL của CloudFront.

Kiểm tra CloudFront Distribution

  1. Sao chép Distribution domain name từ bảng điều khiển CloudFront (ví dụ: d26xve6snn0mdw.cloudfront.net)
  2. Mở một tab trình duyệt mới và truy cập vào URL CloudFront
  3. Xác minh rằng hệ thống Fast Food của bạn hiển thị chính xác
