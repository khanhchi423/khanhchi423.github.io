Configure EventBridge Rules

In this section, we will create an EventBridge rule that supports our Event Sourcing architecture by capturing domain events from our custom event bus. This rule will route order events (OrderPlaced, OrderConfirmed, OrderCancelled) to our Lambda function, which will process these events to update read models and maintain the event store - a key component of Event Sourcing pattern.

Step 1: Navigate to EventBridge Rules

  1. In the Amazon EventBridge console, click Rules in the left navigation panel.
  2. Click Create rule.

Step 2: Define Rule Details

  1. Name: OrderEventProcessingRule
  2. Description: Process all order events
  3. Event bus: Select FastFoodOrderBus from the dropdown
  4. Rule type: Select Rule with an event pattern
  5. Click Next.

Step 3: Build Event Pattern

  1. Creation method: Select Custom pattern (JSON editor)
  2. In the Event pattern text area, enter the following JSON:
{
  "source": ["fastfood.orders"],
  "detail-type": [
    "OrderPlaced",
    "OrderConfirmed", 
    "OrderCancelled"
  ]
}
  1. Verify that JSON is valid indicator shows a green checkmark.
  2. Click Next.

Step 4: Configure Target

  1. Target type: Select AWS service
  2. Select a service: Choose Lambda function
  3. Function: Select OrderEventProcessor from the dropdown
  4. Leave other settings as default.
  5. Click Next.

Step 5: Configure Tags (Optional)

  1. This step is optional. You can add tags to organize and track your EventBridge rule.
  2. Click Next to proceed.

Step 6: Review and Create

  1. Review all the configuration details:

    • Rule name: OrderEventProcessingRule
    • Event bus: FastFoodOrderBus
    • Event pattern: Captures fastfood.orders events
    • Target: OrderEventProcessor Lambda function

  2. Click Create rule.

✅ Result: The EventBridge rule is successfully created and will automatically route order events from the FastFoodOrderBus to the OrderEventProcessor Lambda function for processing.

Create StateReconstructionRule

Now we’ll create a second rule for handling state reconstruction requests.

Step 7: Create Second Rule

  1. Click Create rule again to create another rule.
  2. Configure the rule details:
  • Name: StateReconstructionRule
  • Description: Handle state reconstruction requests
  • Event bus: Select FastFoodOrderBus from the dropdown
  • Rule type: Select Rule with an event pattern
  1. Click Next.

Step 8: Build Event Pattern for State Reconstruction

  1. Creation method: Select Custom pattern (JSON editor)
  2. In the Event pattern text area, enter the following JSON:
{
  "source": ["fastfood.orders"],
  "detail-type": [
    "OrderPlaced",
    "OrderConfirmed", 
    "OrderCancelled"
  ]
}

  1. Verify that JSON is valid indicator shows a green checkmark.

  2. Click Next.

Step 9: Configure Target for State Reconstruction

  1. Target type: Select AWS service
  2. Select a service: Choose Lambda function
  3. Function: Select StateReconstructionHandler from the dropdown
  4. Leave other settings as default.
  5. Click Next.

Step 10: Review and Create State Reconstruction Rule

Review all the configuration details:

  1. Rule name: StateReconstructionRule
  2. Event bus: FastFoodOrderBus
  3. Event pattern: Captures fastfood.orders events
  4. Target: StateReconstructionHandler Lambda function

Please copy the Rule ARN from your StateReconstructionRule in your AWS account.
You can find it in the EventBridge console → Click your rule → Rule ARN (highlighted in the red box).

Configure Lambda Function Permissions

After creating the EventBridge rule, we need to ensure our Lambda function has the proper permissions to be invoked by EventBridge.

  1. Navigate to Lambda Console → Functions → StateReconstructionHandler

  2. Go to the Configuration tab → Permissions

  3. Go to Resource-based policy statements → Add permissions

Add EventBridge Invoke Permission

  1. Service: Select EventBridge (CloudWatch Events)
  2. Statement ID: AllowEventBridgeInvokeStateReconstruction
  3. Principal: events.amazonaws.com
  4. Source ARN: arn:aws:events:ap-southeast-1:218451864254:rule/FastFoodOrderBus/StateReconstructionRule
  5. Action: lambda:InvokeFunction**

The ARN below is just an example. Replace it with the ARN of the StateReconstructionRule that you created in your AWS account.

9. Click Save to apply the permissions.

This permission configuration is for the StateReconstructionHandler Lambda function. The setup is the same as in the OrderEventProcessor permission configuration earlier, except for the function name and the EventBridge rule it is linked to.

Event Sourcing Context: This rule is essential for our Event Sourcing implementation. When domain events (OrderPlaced, OrderConfirmed, OrderCancelled) are published to the FastFoodOrderBus, they represent state changes in our system. The Lambda function will process these events to: Store them in the event store (DynamoDB) Update read models for queries Potentially trigger side effects or notifications. This enables us to reconstruct the application state from the sequence of events - the core principle of Event Sourcing.
